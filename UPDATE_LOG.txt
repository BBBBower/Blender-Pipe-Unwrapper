FIXES APPLIED:
================================================================================

1. UV OUTLINE RENDERING FIXED
   Problem: UV outlines were messy with 320+ boundary edges
   Cause: Too many seam edges (160) being marked incorrectly
   Solution:
   - Fixed seam marking to only mark edges along ground plane (Z≈0)
   - Seam now placed at minimum radius (closest to origin)
   - Kept original edge direction for proper boundary extraction
   Result: Clean outlines with 88 boundary edges, 12 seam edges

2. DEGREES PER SEGMENT RELOCATED
   Problem: Degrees info was in yellow box in middle of page
   Requested: Move to title, remove yellow box
   Solution:
   - Updated subtitle to show: "76.2mm OD | 1.5D 90° Bend | 18.0° per segment"
   - Removed yellow information box entirely
   - Cleaner page layout with info in header
   Result: Degrees per segment now in title, yellow box removed

3. SEAM LOCATION UPDATED
   Problem: Seam was on inside of bend radius (confusing)
   Requested: Seam along ground plane, closest to origin
   Solution:
   - Changed seam marking logic to find minimum radius in XY plane
   - Mark edges where Z≈0 and radius≈minimum
   - More intuitive seam placement
   Result: Seam now on ground plane at point closest to origin

4. WRAP THICKNESS INPUT IMPROVED
   Problem: User had to calculate wrap radius manually
   Requested: Input wrap thickness in mm instead
   Solution:
   - Changed "Wrap Radius" to "Wrap Thickness"
   - Default: 6.15mm (6mm fiberglass + 0.15mm stainless)
   - Automatic calculation: wrap_radius = pipe_radius + wrap_thickness
   Result: More intuitive input, automatic radius calculation

5. ANGLE INPUT FIXED
   Problem: Had to enter 5000+ to get 90 degrees
   Cause: subtype='ANGLE' made Blender treat input as radians
   Solution:
   - Removed subtype='ANGLE' from bend_angle property
   - Added degree symbol (°) to field name
   - Increased max range from 180° to 360°
   Result: Enter 90 to get 90 degrees (works correctly now)

6. CONTEXT ERROR FIXED
   Problem: "Operator bpy.ops.object.select_all.poll() failed, context is incorrect"
   Cause: Using bpy.ops.object.select_all() which requires specific context
   Solution:
   - Replaced ALL operator calls with direct API calls
   - In create_pipe_segment: Use direct bpy.data.objects.remove()
   - In unwrap_pipe_segment: Loop through objects for deselection
   - Ensure object mode before any mode changes
   Result: Template generation works reliably from ANY context (object/edit mode)

================================================================================
USAGE NOTES:
================================================================================

To use the updated addon:

1. Open Blender
2. Press 'N' key in 3D viewport
3. Click "Pipe Templates" tab
4. Enter specifications:
   - Pipe OD in mm (e.g., 76.2 for 3")
   - Bend Radius as multiplier (e.g., 1.5 for 1.5D)
   - Bend Angle in degrees (e.g., 90 for 90°)
   - Number of segments (e.g., 5)
   - Wrap thickness in mm (e.g., 6.15)
   - Overlap in mm (e.g., 10)
5. Click "Generate Template"
6. Open PDF from output folder
7. Print at 100% scale
8. Verify 100mm scale bar
9. Cut along red outline

================================================================================
TROUBLESHOOTING:
================================================================================

If you encounter issues:

1. Angle not working correctly:
   - Make sure you enter the value in degrees (e.g., 90 for 90°)
   - Don't use radians or other units

2. Context error when generating:
   - Addon now handles this automatically
   - Should work from any mode or context

3. UV outline looks messy:
   - This has been fixed in v1.1.0
   - Reload the addon to get the fix

4. Wrap dimensions seem wrong:
   - Check wrap thickness value
   - Should be total thickness (all layers combined)
   - Example: 6mm fiberglass + 0.15mm stainless = 6.15mm

5. PDF doesn't show proper scale:
   - Print at 100% scale (Actual Size)
   - Verify 100mm scale bar with ruler
   - If wrong, check printer settings
